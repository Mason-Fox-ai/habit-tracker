# 🎯 Трекер привычек (Habit Tracker Service)

## 📋 О проекте
Веб-приложение для помощи пользователям в формировании и поддержании полезных привычек. Позволяет создавать список привычек, отмечать выполнение ежедневно, отслеживать серии (streaks) и анализировать прогресс через статистику.

## 🎯 Цель разработки
Разработать концепцию и техническое задание для веб-приложения «Трекер привычек».

## 📁 Структура репозитория
- `/mockups` - макеты интерфейса (изображения)
- `/docs` - документация проекта
- `README.md` - основной файл с техническим заданием

## 1. АНАЛИЗ ПРЕДМЕТНОЙ ОБЛАСТИ

### 1.1 Описание предметной области
В современном ритме жизни люди стремятся к саморазвитию: заниматься спортом, читать, пить воду, медитировать. Однако без системного подхода полезные действия быстро забываются или откладываются.

**Проблемы текущей ситуации:**
- Отсутствие системного подхода к саморазвитию
- Забывание о ежедневных задачах (спорт, чтение, вода)
- Отсутствие визуализации прогресса и мотивации
- Хаос в заметках и бумажных ежедневниках

### 1.2 Целевая аудитория
| Роль | Потребности |
|------|------------|
| **Пользователь** | Вести учёт привычек, видеть прогресс, получать мотивацию |
| **Премиум-пользователь** | Расширенная статистика, неограниченное число привычек |
| **Администратор** | Управление пользователями, модерация контента |

### 1.3 Решаемые задачи
- Автоматизация учёта выполнения ежедневных задач
- Визуализация длительности серий (streaks)
- Анализ продуктивности по категориям
- Централизованное хранение истории привычек

### 2.1 Функциональные требования

#### Роль «Пользователь»
| ID | Требование | Описание |

| F06 | Настройки профиля | Изменение пароля, имени, настроек уведомлений |

#### Роль «Администратор»
| ID | Требование | Описание |
|----|-----------|----------|
| A01 | Управление пользователями | Блокировка нарушителей, просмотр списка пользователей |
| A02 | Модерация | Проверка контента (если есть публичные привычки) |
| A03 | Системная статистика | Общая активность пользователей на платформе |

| N03 | Доступность | Работа через браузер и PWA | Chrome, Safari, Mobile |

## 4. МОДЕЛЬ ДАННЫХ

### 4.1 ER-диаграмма (описание связей)
```
┌─────────────────┐       ┌─────────────────┐       ┌─────────────────┐
│     Users       │       │     Habits      │       │    HabitLogs    │
│   (Пользователи)│       │   (Привычки)    │       │  (История)      │
├─────────────────┤       ├─────────────────┤       ├─────────────────┤
│ id (PK)         │◄──────┤ userId (FK)     │       │ id (PK)         │
│ email           │       │ title           │       │ habitId (FK)    │
│ passwordHash    │       │ color           │       │ date            │
│ fullName        │       │ frequency       │       │ status          │
│ createdAt       │       │ createdAt       │       │ createdAt       │
└─────────────────┘       └─────────────────┘       └─────────────────┘
```

### 4.2 Описание таблиц

**Таблица: Users (Пользователи)**
| Поле | Тип | Описание | Ограничения |
|------|-----|----------|-------------|
| id | INTEGER | Уникальный идентификатор | PRIMARY KEY, AUTO_INCREMENT |
| email | VARCHAR(100) | Email пользователя | UNIQUE, NOT NULL |
| passwordHash | VARCHAR(255) | Хеш пароля | NOT NULL |
| fullName | VARCHAR(100) | Полное имя | NOT NULL |
| createdAt | TIMESTAMP | Дата регистрации | DEFAULT NOW() |

**Таблица: Habits (Привычки)**
| Поле | Тип | Описание | Ограничения |
|------|-----|----------|-------------|
| id | INTEGER | Уникальный идентификатор | PRIMARY KEY, AUTO_INCREMENT |
| userId | INTEGER | ID владельца | FOREIGN KEY (Users.id) |
| title | VARCHAR(100) | Название привычки | NOT NULL |
| color | VARCHAR(20) | Цвет метки (hex) | DEFAULT '#3498db' |
| frequency | ENUM | Частота (daily/weekly) | DEFAULT 'daily' |
| createdAt | TIMESTAMP | Дата создания | DEFAULT NOW() |

**Таблица: HabitLogs (История выполнений)**
| Поле | Тип | Описание | Ограничения |
|------|-----|----------|-------------|
| id | INTEGER | Уникальный идентификатор | PRIMARY KEY, AUTO_INCREMENT |
| habitId | INTEGER | ID привычки | FOREIGN KEY (Habits.id) |
| date | DATE | Дата выполнения | NOT NULL |
| status | BOOLEAN | Выполнено или нет | DEFAULT true |
| createdAt | TIMESTAMP | Время отметки | DEFAULT NOW() |

**Проверка:**
- Уникальность пары `(habitId, date)` — нельзя отметить одну привычку дважды за день
- `date` ≤ сегодня — нельзя отмечать привычки в будущем
- CASCADE при удалении пользователя — удаляются его привычки и логи

## 5. ПРОЕКТИРОВАНИЕ API

### 5.1 REST API Endpoints

#### 🔐 **Аутентификация**
| Метод | Endpoint | Описание | Тело запроса | Ответ |
|-------|---------|----------|--------------|-------|
| POST | `/api/auth/login` | Вход в систему | `{ "email": "user@mail.com", "password": "***" }` | `{ "token": "jwt_token", "user": {...} }` |
| POST | `/api/auth/logout` | Выход | - | `{ "success": true }` |
| POST | `/api/auth/register` | Регистрация | `{ "email": "...", "password": "...", "fullName": "..." }` | `{ "token": "...", "user": {...} }` |

#### 🎯 **Привычки (Habits)**
| Метод | Endpoint | Описание | Параметры/Тело | Ответ |
|-------|---------|----------|---------------|-------|
| GET | `/api/habits` | Список моих привычек | - | Массив привычек |
| GET | `/api/habits/:id` | Детали привычки | - | Объект привычки |
| POST | `/api/habits` | Создать привычку | `{ "title": "Чтение", "color": "#ff0000" }` | Созданная привычка |
| PUT | `/api/habits/:id` | Обновить привычку | `{ "title": "Чтение книг" }` | Обновлённая привычка |
| DELETE | `/api/habits/:id` | Удалить привычку | - | `{ "success": true }` |

#### ✅ **Логи выполнений (Logs)**
| Метод | Endpoint | Описание | Параметры/Тело | Ответ |
|-------|---------|----------|---------------|-------|
| POST | `/api/logs` | Отметить выполнение | `{ "habitId": 5, "date": "2024-12-25" }` | `{ "success": true, "streak": 5 }` |
| DELETE | `/api/logs/:id` | Отменить отметку | - | `{ "success": true }` |
| GET | `/api/logs/history` | История по привычке | `?habitId=5&period=month` | Массив записей |
