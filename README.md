## 📖 Описание проекта

**Назначение:** Веб-приложение для формирования и поддержания полезных привычек с отслеживанием прогресса и статистики.

**Проблемы, которые решает:**
- Отсутствие системного подхода к саморазвитию
- Забывание о ежедневных задачах
- Отсутствие визуализации прогресса
- Хаос в заметках и бумажных ежедневниках

**Решения:**
- ✅ Централизованный список привычек
- ✅ Геймификация (подсчет серий дней подряд)
- ✅ Наглядная статистика и графики
- ✅ Напоминания и уведомления

## 🗄 Модель данных

### ER-диаграмма

![ER Diagram](./docs/images/er-diagram.png)

```
┌─────────────────┐       ┌─────────────────┐       ┌─────────────────┐
│     Users       │       │     Habits      │       │    HabitLogs    │
│   (Пользователи)│       │   (Привычки)    │       │  (История)      │
├─────────────────┤       ├─────────────────┤       ├─────────────────┤
│ id (PK)         │◄──────┤ userId (FK)     │       │ id (PK)         │
│ email           │       │ title           │       │ habitId (FK)    │
│ passwordHash    │       │ color           │       │ date            │
│ fullName        │       │ frequency       │       │ status          │
│ createdAt       │       │ createdAt       │       │ createdAt       │
└─────────────────┘       └─────────────────┘       └─────────────────┘
```

### Таблица: Users

| Поле | Тип | Описание | Ограничения |
| --- | --- | --- | --- |
| id | INTEGER | Уникальный идентификатор | PRIMARY KEY |
| email | VARCHAR(100) | Электронная почта | UNIQUE, NOT NULL |
| passwordHash | VARCHAR(255) | Хеш пароля | NOT NULL |
| fullName | VARCHAR(100) | Полное имя | NOT NULL |
| createdAt | TIMESTAMP | Дата регистрации | DEFAULT NOW() |

### Таблица: Habits

| Поле | Тип | Описание | Ограничения |
| --- | --- | --- | --- |
| id | INTEGER | Уникальный идентификатор | PRIMARY KEY |
| userId | INTEGER | ID владельца | FOREIGN KEY |
| title | VARCHAR(100) | Название привычки | NOT NULL |
| color | VARCHAR(20) | Цвет метки | DEFAULT '#3498db' |
| frequency | ENUM | Частота | DEFAULT 'daily' |

### Таблица: HabitLogs

| Поле | Тип | Описание | Ограничения |
| --- | --- | --- | --- |
| id | INTEGER | Уникальный идентификатор | PRIMARY KEY |
| habitId | INTEGER | ID привычки | FOREIGN KEY |
| date | DATE | Дата выполнения | NOT NULL |
| status | BOOLEAN | Выполнено | DEFAULT true |

## 🌐 API

### Аутентификация

| Метод | Endpoint | Описание |
| --- | --- | --- |
| POST | `/api/auth/login` | Вход в систему |
| POST | `/api/auth/register` | Регистрация |
| POST | `/api/auth/logout` | Выход |

### Привычки

| Метод | Endpoint | Описание |
| --- | --- | --- |
| GET | `/api/habits` | Список привычек |
| POST | `/api/habits` | Создать привычку |
| PUT | `/api/habits/:id` | Обновить привычку |
| DELETE | `/api/habits/:id` | Удалить привычку |

### Логи

| Метод | Endpoint | Описание |
| --- | --- | --- |
| POST | `/api/logs` | Отметить выполнение |
| DELETE | `/api/logs/:id` | Отменить отметку |
| GET | `/api/stats` | Получить статистику |

### Пример запроса

```json
// POST /api/logs
{
  "habitId": 12,
  "date": "2024-12-25"
}

// Ответ (201 Created):
{
  "id": 501,
  "habitId": 12,
  "date": "2024-12-25",
  "status": true,
  "currentStreak": 7
}
```

## 🚀 Установка и запуск

### Требования

- Node.js 18+
- PostgreSQL 14+
- npm или yarn

### Backend

```bash
cd backend
npm install
cp .env.example .env
npm run migrate
npm run dev
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

### Переменные окружения

```env
# Backend
DATABASE_URL=postgresql://user:pass@localhost:5432/habit_tracker
JWT_SECRET=your_secret_key
PORT=5000

# Frontend
REACT_APP_API_URL=http://localhost:5000/api
```

## 📌 Как добавить PlantUML диаграммы в README

GitHub **не поддерживает PlantUML напрямую**, поэтому есть 3 варианта:

### Вариант 1: Экспорт в PNG/SVG (рекомендуется)

1.  Откройте [PlantUML Server](https://www.plantuml.com/plantuml/)
2.  Вставьте код диаграммы
3.  Скачайте как PNG/SVG
4.  Положите в папку `docs/images/`
5.  Вставьте в README:

```markdown
![ER Diagram](./docs/images/er-diagram.png)
```

### Вариант 2: Использовать плагин для GitHub

Добавьте в начало README:

```markdown
<!-- PlantUML support via plugin -->
```

Используйте сервис: [plantuml.com](https://www.plantuml.com/plantuml/)

### Вариант 3: Текстовый формат (как в примере выше)

Просто вставьте ASCII-диаграмму в код-блок:
